## 🚀 Deployment & Operations

### 🔧 Prerequisites
Ensure your local environment is equipped with the core binaries. You will need [Node.js](https://nodejs.org/) and [npm](https://www.npmjs.com/) installed on your machine.

If you lack the global Angular CLI, execute the following directive:
```bash
npm install -g @angular/cli
```

### 📦 Install Dependencies
Navigate to the project sector in your terminal and initialize the dependency matrix:
```bash
npm install
```

### 🌐 Development Server
Boot up the dev server. The application matrix will automatically reload upon detecting source file modifications.
```bash
ng serve
```
*Access the uplink at: `http://localhost:4200/`*

### 🏗️ Code Scaffolding
Generate new modular components on the fly:
```bash
ng generate component <component-name>
```
*Supported Schematics: `directive` | `pipe` | `service` | `class` | `guard` | `interface` | `enum` | `module`*

### 🔨 Build Protocol
Compile the project into the deployment directory. Artifacts will be stored securely in `dist/`.
```bash
ng build
```

## 🧪 Testing & Diagnostics

Powered by **Jasmine** and **Karma** to ensure 100% operational integrity and fault tolerance.

### Unit Testing
Launch the Karma test runner in watch mode. Tests will re-execute upon saving changes.
```bash
ng test
```
For a single, isolated execution without watch mode:
```bash
ng test --watch=false
```

### Deep Debugging
When tests fail, analyze the terminal output or utilize `console.log` statements. For advanced visual debugging within a browser environment:
```bash
ng test --browsers=ChromeDebug
```
*This keeps the browser instance active, allowing full access to developer tools.*

### End-to-End (E2E) Testing
Execute E2E tests via your platform of choice. *(Note: Requires integration of an e2e testing package).*
```bash
ng e2e
```
